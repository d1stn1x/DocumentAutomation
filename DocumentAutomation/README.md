# Система автоматизации создания документов

## Описание

Графическое приложение на Windows Forms для автоматизации создания документов на основе шаблонов с интеграцией MS SQL Server.

## Возможности

### Управление шаблонами
- **Создание, редактирование и удаление шаблонов** через удобный интерфейс
- **Категоризация шаблонов** для организации по типам
- **Хранение в базе данных** MS SQL Server
- **Вставка переменных** в шаблоны одним кликом

### Работа с документами
- **Автоматическое определение переменных** в шаблонах (формат `{{VARIABLE_NAME}}`)
- **Генерация документов** с заполнением переменных пользовательскими данными
- **История документов**: Автоматическое сохранение всех созданных документов
- **Экспорт**: Сохранение готовых документов в текстовые файлы

### Категории и организация
- **Управление категориями** с цветовым кодированием
- **Фильтрация шаблонов** по категориям
- **Описания** для шаблонов и категорий

### Интерфейс
- **Современный дизайн** с цветовым оформлением
- **Удобная навигация** через меню
- **Просмотр истории** созданных документов
- **Встроенные шаблоны**: Примеры для договоров, счетов и писем

## Системные требования

- .NET 10.0 или выше
- Windows (для запуска Windows Forms приложений)
- MS SQL Server или LocalDB

## Открытие в Visual Studio

1. Откройте файл **DocumentAutomation.sln** в Visual Studio 2022 или новее
2. Проект автоматически загрузится со всеми зависимостями
3. Восстановите NuGet пакеты (обычно происходит автоматически)

## Настройка базы данных

### Вариант 1: Использование LocalDB (рекомендуется для разработки)

LocalDB устанавливается вместе с Visual Studio. Никаких дополнительных действий не требуется - база данных создастся автоматически при первом запуске.

Строка подключения по умолчанию в `appsettings.json`:
```json
"ConnectionStrings": {
  "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=DocumentAutomationDB;Trusted_Connection=True;MultipleActiveResultSets=true"
}
```

### Вариант 2: Использование SQL Server Express

1. Установите SQL Server Express с [официального сайта Microsoft](https://www.microsoft.com/sql-server/sql-server-downloads)
2. Измените строку подключения в `appsettings.json`:

```json
"ConnectionStrings": {
  "DefaultConnection": "Server=.\\SQLEXPRESS;Database=DocumentAutomationDB;Trusted_Connection=True;MultipleActiveResultSets=true"
}
```

### Вариант 3: Использование полного SQL Server

Измените строку подключения в `appsettings.json` на свою:

```json
"ConnectionStrings": {
  "DefaultConnection": "Server=ИМЯ_СЕРВЕРА;Database=DocumentAutomationDB;User Id=ИМЯ_ПОЛЬЗОВАТЕЛЯ;Password=ПАРОЛЬ;MultipleActiveResultSets=true"
}
```

## Использование приложения

### Главное окно

После запуска приложения вы увидите главное окно с тремя основными областями:

1. **Левая панель**: Список доступных шаблонов из базы данных
2. **Центральная/правая панель**: Поля для ввода значений переменных
3. **Нижняя панель**: Отображение сгенерированного документа

### Управление шаблонами

**Меню "Шаблоны" → "Управление шаблонами"**

- **Добавить**: Создать новый шаблон
  - Введите название шаблона
  - Добавьте описание (необязательно)
  - Выберите категорию (необязательно)
  - Введите содержимое с переменными в формате `{{ИМЯ_ПЕРЕМЕННОЙ}}`
  - Используйте кнопку "Вставить переменную" для быстрой вставки

- **Редактировать**: Изменить существующий шаблон
  - Дважды кликните на шаблоне в списке или выберите и нажмите "Редактировать"
  - Внесите изменения и нажмите "Сохранить"

- **Удалить**: Удалить шаблон
  - Выберите шаблон и нажмите "Удалить"
  - Подтвердите удаление

### Управление категориями

**Меню "Шаблоны" → "Управление категориями"**

- **Добавить категорию**: Создать новую категорию для организации шаблонов
  - Введите название категории
  - Добавьте описание (необязательно)
  - Укажите цвет в формате HTML (например: #FFE6E6)

- **Удалить категорию**: Удалить категорию (шаблоны в категории станут без категории)

### Создание документов

1. **Выберите шаблон** из списка слева
2. **Заполните переменные** в центральной панели (автоматически создаются поля для всех переменных)
3. **Нажмите "Сгенерировать документ"** - документ появится в нижней панели и автоматически сохранится в базу данных
4. **Нажмите "Сохранить документ"** для экспорта в файл (необязательно)

### Просмотр истории

**Меню "Шаблоны" → "История документов"**

- Просмотр всех созданных документов
- Дважды кликните на документе для просмотра содержимого
- Используйте кнопку "Экспорт" для сохранения документа в файл

## Примеры переменных

В шаблонах используйте двойные фигурные скобки для обозначения переменных:

```
{{CLIENT_NAME}}      - Имя клиента
{{DATE}}             - Дата (автоматически заполняется текущей)
{{CONTRACT_NUMBER}}  - Номер договора
{{PRICE}}            - Цена
{{COMPANY_NAME}}     - Название компании
```

## Структура базы данных

### Таблица Templates
- **Id** (int, PK) - Идентификатор шаблона
- **Name** (nvarchar(200)) - Название шаблона
- **Content** (nvarchar(max)) - Содержимое шаблона
- **Description** (nvarchar(500), nullable) - Описание шаблона
- **CreatedDate** (datetime2) - Дата создания
- **ModifiedDate** (datetime2, nullable) - Дата изменения

### Таблица GeneratedDocuments
- **Id** (int, PK) - Идентификатор документа
- **DocumentName** (nvarchar(200)) - Название документа
- **Content** (nvarchar(max)) - Содержимое документа
- **GeneratedDate** (datetime2) - Дата генерации
- **TemplateId** (int, FK) - Ссылка на шаблон

## Работа с приложением

### Главное окно

- **Левая панель**: Список доступных шаблонов из БД
- **Правая верхняя панель**: Поля для ввода значений переменных
- **Нижняя панель**: Отображение сгенерированного документа
- **Кнопки управления**:
  - "Сгенерировать документ" - создание документа на основе заполненных данных (автоматически сохраняется в БД)
  - "Сохранить документ" - экспорт результата в файл
  - "Обновить список" - перезагрузка списка шаблонов из БД

### Работа с шаблонами

Шаблоны хранятся в таблице **Templates** базы данных. Шаблон представляет собой текст с переменными в формате `{{VARIABLE_NAME}}`.

**Пример шаблона договора:**
```
ДОГОВОР №{{CONTRACT_NUMBER}}

Дата: {{DATE}}

Между {{COMPANY_NAME}} (Исполнитель) и {{CLIENT_NAME}} (Заказчик)

Предмет договора: {{CONTRACT_SUBJECT}}
```

## Встроенные шаблоны

При первом запуске автоматически создаются три примера шаблонов:

1. **Договор** - шаблон договора
2. **Счет** - шаблон счета на оплату
3. **Письмо** - шаблон делового письма

## Возможные проблемы и решения

### "Не удается подключиться к базе данных"
- Проверьте, что SQL Server запущен
- Убедитесь, что строка подключения в appsettings.json корректна
- Для LocalDB: убедитесь, что она установлена (входит в Visual Studio)

### "LocalDB не найден"
Установите LocalDB:
- Через Visual Studio Installer → Individual Components → SQL Server Express LocalDB
- Или скачайте отдельно с сайта Microsoft

### База данных не создается
Приложение автоматически создаст базу данных при первом запуске. Если возникли проблемы:
1. Проверьте права доступа к SQL Server
2. Попробуйте запустить Visual Studio от имени администратора
